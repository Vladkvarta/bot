Шпаргалка по управлению проектомЗдесь собрана вся основная информация о структуре вашего проекта, файлах и командах для управления и диагностики.1. Структура проекта и ключевые файлыВсе файлы вашего проекта находятся на Orange Pi в папке ~/telegram-app.index.js: Сердце вашего приложения. Содержит логику Telegram-бота, веб-сервер Express и обработчик вебхуков от GitHub./public/: Папка для фронтенда. Все файлы, которые вы сюда положите (index.html, style.css, script.js), будут доступны на вашем сайте.options.json: Ваша "база данных" товаров. Редактируйте этот файл, чтобы менять ассортимент, цены и описания..env: Секретные настройки. Хранит токен бота, URL веб-приложения и секрет для вебхука. Никогда не добавляйте этот файл в Git.Редактировать: nano ~/telegram-app/.envecosystem.config.js: Конфигурация для PM2. Указывает, как запускать ваше приложение (имя процесса telegram-bot, главный файл index.js).deploy.sh: Скрипт автоматического обновления на сервере. Запускается после git push и выполняет все шаги по развертыванию.Посмотреть/Редактировать: nano ~/telegram-app/deploy.sh2. Основной рабочий процесс (на вашем ПК)Это цикл действий, который вы выполняете для обновления вашего сайта.Пишете код: Вносите изменения в файлы проекта на вашем локальном компьютере.Сохраняете изменения в Git:# Добавить все измененные файлы
git add .

# Сделать коммит с описанием
git commit -m "Краткое описание ваших изменений"

# Отправить на GitHub
git push
После команды git push обновление на сервере произойдет автоматически.3. Команды для управления и диагностики (на Orange Pi)Эти команды нужно выполнять, подключившись к вашему серверу по SSH (ssh admin@ssh.smakuie.com).Посмотреть статус приложения:pm2 status
(Показывает, запущен ли процесс telegram-bot и его состояние).Посмотреть логи в реальном времени:pm2 logs telegram-bot
(Самая важная команда для поиска ошибок. Чтобы выйти, нажмите Ctrl+C).Перезапустить приложение вручную:pm2 restart telegram-bot
(Нужно, если вы внесли изменения прямо на сервере, например, в файле .env).Проверить, что скрипт обновления отработал:Зайдите в Settings -> Webhooks в вашем репозитории на GitHub.Убедитесь, что у последней доставки (Recent Deliveries) стоит зелёная галочка.
